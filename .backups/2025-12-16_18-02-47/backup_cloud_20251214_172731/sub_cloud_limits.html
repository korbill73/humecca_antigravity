서비스별 제한사항

KT Cloud 서비스를 이용하시기 전에 아래 제한사항을 확인하시기 바랍니다. 제한사항은 서비스 안정성과 공정한 자원 배분을 위해 설정되어 있습니다.

Server 서비스 제한사항
항목 기본 제한 확장 요청 시
VM 최대 생성 수 계정당 50대 요청 시 확장 가능
vCPU 최대 수 VM당 32 vCore -
Memory 최대 용량 VM당 256GB -
Data Disk 최대 용량 VM당 2TB 추가 디스크 연결 가능
Public IP 개수 계정당 10개 요청 시 추가 할당
Database 서비스 제한사항
항목 기본 제한 비고
DB 인스턴스 수 계정당 20개 요청 시 확장
스토리지 최대 용량 인스턴스당 1TB -
동시 연결 수 사양에 따라 다름 MySQL 기준 150~1000
백업 보관 기간 최대 30일 자동 백업 기준
Storage/CDN 서비스 제한사항
항목 기본 제한 비고
Object Storage 용량 계정당 100TB 요청 시 확장
단일 파일 용량 5TB 멀티파트 업로드 사용 시
CDN 도메인 수 계정당 20개 -
CDN 캐시 갱신 1,000 URL/일 Purge 요청 기준
Network 서비스 제한사항
항목 기본 제한 비고
VPC 개수 계정당 5개 Zone별
Subnet 개수 VPC당 10개 -
Load Balancer 계정당 10개 요청 시 추가
방화벽 규칙 그룹당 100개 Inbound + Outbound

참고사항

• 위 제한사항은 KT Cloud 정책에 따라 변경될 수 있습니다.

• 추가 자원이 필요한 경우 고객센터(02-418-7766)로 문의해 주세요.

• 일부 서비스는 Zone에 따라 제한사항이 다를 수 있습니다.

• 자세한 내용은 KT Cloud 공식 문서를 참조하시기 바랍니다.
